# thesis-review-skill

本科开发类毕业论文审核技能，适用于 [Claude Code](https://claude.ai/code)。

基于 12 篇论文的实战审核经验打磨，能够对开发类本科毕业论文进行多维度自动化审核，生成结构化审核报告，并直接在 Word 文档中添加批注。

## 定位

面向**开发类本科毕业论文**（如基于 SpringBoot + Vue 的 XX 管理系统），提供标准化的审核流程。

**审核标准定位**：

| 维度 | 态度 | 说明 |
|------|------|------|
| 结构完整性 | 严格 | 7 章必选节逐一验证 |
| 逻辑自洽性 | 严格 | 前后矛盾、数据不一致必须指出 |
| 格式规范性 | 严格 | 图表编号、标点、参考文献格式 |
| 明显错误 | 严格 | 错别字、技术概念错误、复制粘贴痕迹 |
| 内容一致性 | 适度 | 需求-设计-实现-测试链路对应 |
| 研究深度 | 宽松 | 不苛求创新性和文献综述广度 |

**支持两种审核模式**：
- **初稿审核**：检查结构(R)、一致性(C)、格式(F)、逻辑(L) 四个维度
- **修改稿审核**：在初稿基础上额外启用深度审核(D)，逐段检查内容准确性、语言规范性、表述专业性

## 实现原理

### 审核规则体系

共 5 大类、30+ 条审核规则，定义在 [`references/rules.md`](references/rules.md) 中：

```
R（结构规则）    ─── R1-R7：绪论、技术、需求、设计、实现、测试、总结 共 7 章
C（一致性规则）  ─── C1-C6：名称、技术栈、链路、设计、图表、数据 一致性
F（格式规则）    ─── F1-F6：图表、标题、标点、人称、参考文献、代码 格式
L（逻辑规则）    ─── L1-L5：逻辑自洽、内容完整、明显错误、复制粘贴、编号
D（深度规则）    ─── D1-D3：内容准确性、语言规范性、表述专业性
```

### 技术栈

- **论文解析**：`pandoc` 将 `.docx` 转为 Markdown，便于 AI 阅读和分析
- **文档操作**：基于 Claude Code 的 [docx skill](https://github.com/anthropics/anthropic-agent-skills) 进行 OOXML 级别的文档操作
- **批注注入**：解包 `.docx` → 解析 `document.xml` 定位段落 → 调用 `Document.add_comment()` 添加批注 → 重新打包
- **环境管理**：`uv` 创建项目本地虚拟环境，安装 `defusedxml` 和 `lxml`

### 工作流程

```
论文.docx
    │
    ├── pandoc ──→ 论文.md ──→ AI 逐维度审核 ──→ 审核报告.md
    │
    └── unpack ──→ document.xml
                      │
                      ├── 关键词搜索定位段落索引
                      │
                      ├── 批量添加批注（Python 脚本）
                      │
                      └── pack ──→ 论文-已审核.docx（含批注）
```

### 高频问题知识库

内置基于 12 篇论文实战总结的高频问题清单（按发现频率排序），指导 AI 重点关注：

1. 复制粘贴错误（10/12 篇）—— 内容来自其他论文未修改、截图重复引用
2. 缺少独立技术章节（7/12 篇）
3. 章节/图表编号错误（8/12 篇）
4. 错别字和基本语言错误（10/12 篇）—— "登陆"→"登录"、RSA 被称为"对称加密"等
5. 缺少关键章节（7/12 篇）—— 非功能需求、业务流程设计、架构图等

## 使用指南

### 前置要求

- [Claude Code](https://claude.ai/code) CLI 或桌面端
- `pandoc`（用于 docx 转 markdown）
- `uv` 或 `pip`（用于创建 Python 虚拟环境）
- Claude Code 的 [docx skill](https://github.com/anthropics/anthropic-agent-skills)（用于操作 Word 文档）

### 安装

将本仓库克隆到 Claude Code 的 skills 目录：

```bash
git clone https://github.com/thinking-bzf/thesis-review-skill.git \
  ~/.claude/skills/thesis-review
```

### 基本用法

在 Claude Code 中直接说：

```
帮我审核一下这篇论文：/path/to/论文.docx
```

或使用 skill 命令：

```
/thesis-review /path/to/论文.docx
```

AI 会依次询问：
1. **批注作者姓名** —— 在 Word 批注中显示的署名
2. **论文版本** —— 初稿 or 修改稿（决定是否启用 D 类深度审核规则）

### 批量审核

一次审核多篇论文：

```
帮我审核 论文目录/ 里的所有论文
```

推荐流程：
1. 主线程统一完成环境准备（venv、pandoc 转换、docx 解包）
2. 并行启动 agent 生成各篇审核报告
3. 主线程串行完成 Word 批注添加和打包
4. 统一打包交付

### 输出产物

每篇论文生成两个文件：

| 文件 | 说明 |
|------|------|
| `论文名-审核报告.md` | 结构化审核报告，含 R/C/F/L/D 五维度检查结果、问题汇总（按严重程度分类）、审核总结 |
| `论文名-已审核.docx` | 原始 Word 文档 + 审核批注，可直接在 Word/WPS 中打开查看 |

**审核报告结构**：

```markdown
# 论文审核报告

一、结构检查结果（R）    ─── 各章节是否存在、内容是否达标
二、一致性检查结果（C）  ─── 跨章节内容比对
三、格式检查结果（F）    ─── 图表、编号、标点、参考文献
四、逻辑检查结果（L）    ─── 前后矛盾、错别字、复制粘贴
五、深度审核结果（D）    ─── 内容准确性、语言规范性、表述专业性
六、问题汇总             ─── [必须修改] / [建议修改] / [可选改进]
七、审核总结             ─── 总体评价 + 优先修改建议
```

**批注格式**：`[规则编号] 问题描述和修改建议`

示例：
- `[R2.1] 论文缺少独立的"相关技术与工具"章节，建议增设...`
- `[L3.1] "登陆"应为"登录"，请全文统一修改`
- `[D1.4] "RSA对称加密"是基本技术概念错误，RSA是非对称加密算法`

### 自定义规则

审核规则定义在 [`references/rules.md`](references/rules.md) 中，可根据需要修改：

- 调整某条规则为必选/可选
- 新增特定领域的检查项
- 修改判定标准的松紧度
